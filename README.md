El objetivo principal fue construir una base sólida en estructuras de datos, lógica de control de flujo y modularización, preparando el terreno antes de dar el salto a la automatización real sobre entornos Linux.

El progreso técnico se estructuró a través de tres pilares clave:

Control de Flujo Avanzado: Manipulación limpia de bucles while y for, optimizando el uso de break para romper ciclos de forma eficiente y evitar redundancias de código al capturar entradas del usuario o simular bloqueos de seguridad.

Estructuras de Datos Nativas: Implementación práctica de listas para el manejo de historiales de eventos, uso del operador in para filtrado rápido de datos (como IPs o elementos en listas negras) y mapeo de configuraciones o roles mediante diccionarios clave-valor.

Modularización (Principio DRY): Creación de funciones personalizadas (def) combinadas con f-strings y retornos dinámicos (return) para procesar datos y formatear reportes de estado o alertas en tiempo real.

Con estos fundamentos totalmente dominados y estructurados, la siguiente fase dejará atrás las simulaciones de datos locales para pasar a la interacción directa con el sistema operativo. El próximo paso será utilizar la librería nativa subprocess para ejecutar comandos reales de la terminal Bash (como ping o ifconfig), capturar sus salidas y automatizar la gestión de infraestructura de red.
